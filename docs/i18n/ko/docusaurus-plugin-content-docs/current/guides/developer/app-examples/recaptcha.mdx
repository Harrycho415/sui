이 가이드는 사용자가 contract와 상호작용하기 전에 사람이며 봇이 아님을 확인하기 위해 reCAPTCHA를 사용하는 Move 스마트 계약을 작성하고 배포하는 방법을 보여준다.
CAPTCHA는 인간임을 증명하기 위해 챌린지 테스트를 통과해야 하는 봇 방지 방법이다.
CAPTCHA 테스트는 봇이 작업을 수행하는 것을 막는 데 효과적이지만 테스트가 너무 어렵거나 너무 자주 나타나면 정당한 사용자에게 성가시거나 좌절감을 줄 수 있다.
reCAPTCHA는 CAPTCHA 테스트의 한 형태이다.

## Move smart contract

모든 Sui 앱과 마찬가지로 체인상의 Move package가 reCAPTCHA module의 로직을 구동한다.
다음 지침은 module을 만들고 게시하는 과정을 안내한다.

### reCAPTCHA module

시작하기 전에 Move package를 초기화해야 한다.
예시를 저장하려는 디렉터리에서 terminal 또는 console을 열고 다음 명령을 실행해 `recaptcha`라는 이름의 빈 package를 만든다:

```sh
$ sui move new recaptcha
```

이제 코드를 살펴볼 차례이다.
`sources` 디렉터리에 `recaptcha.move`라는 이름의 새 파일을 만들고 다음 코드로 파일을 채운다:

```rust title='recaptcha.move'
// Copyright (c) Mysten Labs, Inc.
// SPDX-License-Identifier: Apache-2.0

module recaptcha::recaptcha {
    // vector를 조작하기 위해 vector module을 import한다.
    use std::vector;

    // 현재 시간을 가져오기 위해 clock module을 import한다.
    use sui::clock::Clock;
    // object에 custom field를 추가하기 위해 dynamic_field module을 import한다.
    use sui::dynamic_field as df;
    // 서명을 검증하기 위해 ed25519 module을 import한다.
    use sui::ed25519::ed25519_verify;
    // 이벤트를 발생시키기 위해 event module을 import한다.
    use sui::event::emit;
    // 수학 연산을 수행하기 위해 math module을 import한다.
    use sui::math;
    // object를 생성하고 조작하기 위해 object module을 import한다.
    use sui::object::{Self, UID};
    // object를 공유하고 전송하기 위해 transfer module을 import한다.
    use sui::transfer;
    // transaction 정보에 접근하기 위해 tx_context module을 import한다.
    use sui::tx_context::{sender, TxContext};

    /// cooldown 기간을 위반한 transaction을 위한 오류 코드.
    const EVerificationExpired: u64 = 0;
    /// 유효하지 않은 서명을 위한 오류 코드.
    const EInvalidSignature: u64 = 1;
    /// 아직 검증되지 않은 송신자를 위한 오류 코드.
    const ENotYetVerified: u64 = 2;

    // 시간 창의 길이를 밀리초 단위로 나타내는 상수를 정의한다.
    const TIME_WINDOW: u64 = 60_000;
}
```

이 코드에서 유의할 세부 사항은 다음과 같다:

1. 네 번째 줄은 package `recaptcha` 안에 있는 module 이름을 `recaptcha`로 선언한다.
1. 다음 여덟 줄은 `sui::clock::Clock`, `sui::dynamic_field as df`, `sui::ed25519::ed25519_verify`, `sui::event::emit,sui::math`, `sui::object::{Self, UID}`, `sui::transfer`, `sui::tx_context::{sender, TxContext}` 같은 다른 module의 타입과 함수를 import하기 위해 use keyword를 사용한다. 이 module들은 reCAPTCHA 검증과 상호작용 로직을 구현하는 데 필요하다.
1. 이어지는 세 줄은 reCAPTCHA 테스트 결과의 유효성과 사용자의 자격 여부를 확인하는 데 사용하는 `EVerificationExpired`, `EInvalidSignature`, `ENotYetVerified` 오류 코드를 선언한다. 이 오류 코드는 unit test에서 프로그램의 정확성을 검증하는 데에도 사용된다.
1. 마지막 줄은 시간 창의 길이를 밀리초 단위로 지정하는 상수 `TIME_WINDOW`를 선언한다. 시간 창은 사용자가 reCAPTCHA 테스트를 통과한 뒤 스마트 계약과 상호작용할 자격이 있는 기간이다.

다음으로 이 module에 코드를 조금 더 추가한다:

```rust title='recaptcha.move'
struct Interaction has copy, drop {
    sender: address, // 송신자의 address
    timestamp_ms: u64, // 밀리초 단위 timestamp
}

// key field를 가진 registry object를 위한 struct를 정의한다.
struct Registry has key {
    id: UID, // registry object의 고유 식별자
    window: u64, // 밀리초 단위 시간 창의 길이
}

// registry를 초기화하는 함수를 정의한다.
fun init(ctx: &mut TxContext) {
    // registry object를 다른 참여자와 공유한다.
    transfer::share_object(
        Registry {
            id: object::new(ctx), // 고유 id를 가진 새 object를 만든다.
            window: TIME_WINDOW, // 시간 창을 상수 값으로 설정한다.
        }
    );
}
```

- `Interaction` struct는 사용자가 스마트 계약과 성공적으로 상호작용했을 때 이벤트로 발생하는 데이터를 정의하는 데 사용된다. `Interaction` 이벤트는 송신자의 address와 밀리초 단위 timestamp라는 두 필드를 가진다. 송신자의 address는 상호작용을 시작한 계정이고 timestamp는 이벤트가 발생했을 때의 현재 시간이다.
- `Registry` struct는 사용자의 address와 스마트 계약과 상호작용할 자격의 만료 시간 간의 매핑을 저장한다. 또한 밀리초 단위 시간 창의 길이를 지정하는 `window` 필드도 가진다. 시간 창은 사용자가 reCAPTCHA 테스트를 통과한 뒤 스마트 계약과 상호작용할 자격이 있는 기간이다.
- [`init` function](../../../concepts/sui-move-concepts.mdx)은 `Registry`를 위한 [shared object](/guides/developer/objects/object-ownership/shared)를 만든다. 이 함수는 스마트 계약이 블록체인에 배포될 때 호출된다. 이 함수는 고유 ID를 가진 새 registry object를 만들고 시간 창을 상수 값으로 설정한다.

지금까지 module 안의 데이터 구조를 설정했다.
이제 message를 검증하는 함수를 만든다.

```rust title='recaptcha.move'
/// @param registry: registry object.
/// @param signature: Ed25519 타원 곡선의 한 점인 32바이트 서명.
/// @param public_key: Ed25519 타원 곡선의 한 점인 32바이트 공개 키.
/// @param msg: 서명을 검증하는 기준이 되는 message.
public fun verify(
    registry: &mut Registry,
    signature: vector<u8>,
    public_key: vector<u8>,
    msg: vector<u8>,
    ctx: &mut TxContext
) {
    let verified = ed25519_verify(&signature, &public_key, &msg);
    assert!(verified, EInvalidSignature);

    if (!df::exists_with_type<address, u64>(&registry.id, sender(ctx))) {
        df::add<address, u64>(
            &mut registry.id,
            sender(ctx),
            msg_to_ts(&msg)
        );
    } else {
        let timestamp_ms = df::borrow_mut<address, u64>(&mut registry.id, sender(ctx));
        *timestamp_ms = msg_to_ts(&msg);
    }
}

/// 바이트 vector인 message에서 timestamp_ms를 가져와 u64로 변환하는 함수.
public fun msg_to_ts(
    message: &vector<u8>
): u64 {
    let vec_length = vector::length(message);

    let (value, i) = (0u64, 0u8);
    while (i < 13) {
        let element = (*vector::borrow(message, vec_length - (i as u64) - 1) - 48 as u64); // '0'은 48
        value = value + element * math::pow(10, i); // 10^i
        i = i + 1;
    };
    value
}
```

`verify` 함수는 누구나 함수 호출을 통해 자신을 non-bot으로 등록할 수 있게 하는 public 함수이다.
이 함수는 다섯 개의 인자를 받는다:

- `registry`: 사용자의 address와 자격의 만료 시간 간 매핑을 저장하는 registry object.
- `signature`: Ed25519 타원 곡선의 한 점인 32바이트 서명이다. 이 서명은 사용자의 address와 현재 timestamp를 포함하는 message와 오라클의 개인 키를 사용해 오라클이 생성한다.
- `public_key`: Ed25519 타원 곡선의 한 점인 32바이트 공개 키이다. 공개 키는 서명을 검증하는 데 사용하는 오라클의 공개 키이다.
- `msg`: 사용자의 address와 현재 timestamp를 포함하는 message이다. Message는 바이트 vector로 인코딩된다.
- `ctx`: 송신자, 가스 한도, 가스 가격에 대한 정보를 제공하는 transaction context.

이 함수는 다음 단계를 수행한다:

- `sui::ed25519` module의 `ed25519_verify` 함수를 호출해 주어진 공개 키와 message에 대해 서명이 유효한지 확인한다. `ed25519_verify` 함수는 서명의 유효성을 나타내는 boolean 값을 반환한다.
- 서명이 유효하다고 assert하며, 그렇지 않으면 오류 코드 `EInvalidSignature`로 실행을 중단한다.
- 사용자의 address가 registry object의 dynamic field에 key로 존재하는지 확인한다. Dynamic field는 key-value 쌍을 미리 선언하지 않고 object에 저장하는 방법이며 `sui::dynamic_field` module을 사용해 접근, 수정, 삭제할 수 있다.
- 사용자의 address가 dynamic field에 key로 존재하지 않으면 dynamic field에 새 key-value 쌍을 추가한다. Key는 사용자의 address이고 값은 자격의 만료 시간이다. 만료 시간은 message를 밀리초 단위 timestamp로 변환하는 `msg_to_ts` 함수를 호출해 계산한다.
- 사용자의 address가 이미 dynamic field에 key로 존재하면 해당 key의 값을 새로운 자격 만료 시간으로 업데이트한다.

이제 `verify`를 구현했으므로 다음 단계인 누군가가 contract와 상호작용할 수 있는 방식을 보여주는 단계로 넘어갈 수 있다.
사용자가 검증되었는지 아닌지 확인하는 `interact` 함수를 작성한다.

```rust title='recaptcha.move'
// registry object와 상호작용하기 위한 public 함수를 정의한다.
public fun interact(
    registry: &mut Registry,  // registry object에 대한 mutable reference
    clock: &Clock,  // clock object에 대한 reference
    ctx: &mut TxContext  // transaction context에 대한 mutable reference
) {
    // 송신자 address에 대한 기존 상호작용 기록이 registry object에 있는지 확인한다.
    if (df::exists_with_type<address, u64>(&registry.id, sender(ctx))) {
        // 상호작용 기록 object에 대한 mutable reference를 빌린다.
        let timestamp_ms = df::borrow_mut<address, u64>(&mut registry.id, sender(ctx));
        // clock object에서 현재 밀리초 timestamp를 가져온다.
        let current_timestamp = sui::clock::timestamp_ms(clock);

        if (current_timestamp - *timestamp_ms <= registry.window) {
            emit(
                Interaction{
                    sender: sender(ctx),
                    timestamp_ms: sui::clock::timestamp_ms(clock)
                }
            );
        } else {
            abort EVerificationExpired
        }
    } else {
        abort ENotYetVerified
    }
}
```

`interact` 함수는 사용자가 reCAPTCHA 테스트를 통과한 뒤 스마트 계약과 상호작용할 수 있게 하는 public 함수이다.
이 함수는 reCAPTCHA 테스트 결과를 검증하고 사용자가 스마트 계약과 상호작용할 자격을 등록하는 `verify` 함수와 연결된다.
이 함수는 세 개의 인자를 받는다:

- `registry`: 사용자의 address와 자격의 만료 시간 간 매핑을 저장하는 registry object에 대한 mutable reference.
- `clock`: 현재 밀리초 timestamp를 제공하는 clock object에 대한 reference.
- `ctx`: 송신자, 가스 한도, 가스 가격에 대한 정보를 제공하는 transaction context에 대한 mutable reference.

이 함수는 다음 단계를 수행한다:

- 송신자 address에 대한 기존 상호작용 기록이 registry object에 있는지 확인한다. 상호작용 기록은 registry object의 dynamic field로 저장된다. Dynamic field는 key-value 쌍을 미리 선언하지 않고 object에 저장하는 방법이다. Dynamic field는 `sui::dynamic_field` module을 사용해 접근, 수정, 삭제할 수 있다.
- 기존 상호작용 기록이 있으면 상호작용 기록 object에 대한 mutable reference를 빌린다. 상호작용 기록 object에는 밀리초 단위 자격 만료 시간이 들어 있다.
- clock object에서 현재 밀리초 timestamp를 가져와 자격의 만료 시간과 비교한다. 현재 timestamp가 자격의 시간 창 안에 있으면 interaction event를 발생시킨다. Interaction event는 송신자 address와 밀리초 단위 timestamp를 담은 struct이다. Interaction event는 투표, 입찰, 플레이 같은 탈중앙화 애플리케이션의 로직을 구현하는 데 사용할 수 있다.
- 현재 timestamp가 자격의 시간 창 밖에 있으면 오류 코드 `EVerificationExpired`로 실행을 중단한다. 이는 사용자가 스마트 계약과 상호작용하려면 reCAPTCHA 테스트를 다시 통과해야 함을 의미한다.
- 기존 상호작용 기록이 없으면 오류 코드 `ENotYetVerified`로 실행을 중단한다. 이는 사용자가 아직 reCAPTCHA 테스트를 통과하지 않았으며 스마트 계약과 상호작용할 수 없음을 의미한다.

이로써 `recaptcha.move` 코드는 완성된다.

## Backend

reCAPTCHA를 위한 backend를 구현하려면 HTTP request와 response를 처리할 수 있는 `express` 앱을 만들어야 한다.
또한 `@noble/ed25519`, `axios`, `cors`, `helmet`, `morgan`, `dotenv` 같은 의존성도 설치해야 한다.
이 package들은 암호화, HTTP request, cross-origin resource sharing, 보안, 로깅, 환경 변수 처리를 돕는다.

다음은 backend를 만드는 단계이다:

1. `npm init -y`로 새 project를 초기화한다.
2. `npm install --save @noble/ed25519 axios cors helmet morgan dotenv` 또는 `yarn add @noble/ed25519 axios cors helmet morgan dotenv`로 의존성을 설치한다.
3. `app.ts`라는 파일을 만들고 다음 코드를 붙여 넣는다.

```typescript title='app.ts'
import * as ed from '@noble/ed25519';
import axios from 'axios';
import cors from 'cors';
import express from 'express';
import helmet from 'helmet';
import morgan from 'morgan';

import api from './api';
import MessageResponse from './interfaces/MessageResponse';
import * as middlewares from './middlewares';

require('dotenv').config();

const app = express();

app.use(morgan('dev'));
app.use(helmet());
app.use(cors());
app.use(express.json());

app.get<{}, MessageResponse>('/', (req, res) => {
	res.json({
		message: 'Express + TypeScript Server',
	});
});

interface RecaptchaApiResponse {
	success: boolean;
	challenge_ts: string; // 챌린지가 로드된 timestamp(ISO 형식 yyyy-MM-dd'T'HH:mm:ssZZ)
	hostname: string; // reCAPTCHA가 해결된 사이트의 hostname
	signature?: Uint8Array;
	pubKey?: Uint8Array;
	message?: Uint8Array;
	'error-codes'?: any[]; // 선택 사항
}

app.post('/verify-token', async (req, res) => {
	const now: number = Date.now();
	const privKey = process.env.SK!;

	const pubKey = await ed.getPublicKey(privKey);
	const { response, secret, userAddress } = req.body;

	console.log('userAddress: ' + userAddress);
	console.log('secret: ' + secret);
	console.log('response: ' + response);
	console.log('now: ' + now);
	console.log('privKey: ' + privKey);

	const message: string = stringToHex(userAddress.replace('0x', '').concat(now.toString()));

	console.log('message: ' + message);

	const signature = await ed.sign(message, privKey);
	const isValid = await ed.verify(signature, message, pubKey);

	console.log({ message, pubKey, signature, isValid });

	try {
		let axiosResponse = await axios.post<RecaptchaApiResponse>(
			`https://www.google.com/recaptcha/api/siteverify?secret=${secret}&response=${response}`,
		);
		console.log(axiosResponse.data);
		return res.status(200).json({
			success: axiosResponse.data.success,
			verificationInfo: axiosResponse.data,
			signature: Array.from(signature),
			pubKey: Array.from(pubKey),
			message: Array.from(Uint8Array.from(Buffer.from(message, 'hex'))),
		});
	} catch (error) {
		console.log(error);
		return res.status(500).json({
			success: false,
		});
	}
});

function stringToHex(str: string): string {
	let hex = '';
	for (let i = 0; i < str.length; i++) {
		const charCode = str.charCodeAt(i);
		const hexValue = charCode.toString(16);

		// 두 자리 표현을 보장하기 위해 0으로 채운다.
		hex += hexValue.padStart(2, '0');
	}
	return hex;
}

app.use('/api/v1', api);

app.use(middlewares.notFound);
app.use(middlewares.errorHandler);

export default app;
```

4. 코드가 무엇을 하는지 살펴본다.

- 먼저 앱에 필요한 module을 import한다.
- 다음으로 `express` 앱을 만들고 기능을 강화하기 위해 몇 가지 middleware를 사용한다. `morgan`은 로깅에, `helmet`은 보안에, `cors`는 cross-origin resource sharing에, `express.json`은 JSON 데이터 파싱에 사용한다.
- 그다음 루트 경로(`/`)에 대한 `GET` route를 정의하고 단순한 JSON message를 반환한다.
- 그 후 reCAPTCHA API response를 위한 interface를 정의한다. 이는 사용자의 response token을 검증할 때 Google로부터 받는 데이터이다. 여기에는 `success`, `challenge_ts`, `hostname`, `error-codes` 같은 필드가 포함된다. 또한 나중에 추가할 `signature`, `pubKey`, `message` 같은 선택적 필드도 가진다.
- 다음으로 사용자의 response token 검증을 처리하는 `/verify-token` 경로에 대한 `POST` route를 정의한다. 이것이 backend의 핵심 로직이다. 이 route에서 따르는 단계는 다음과 같다:
  - 현재 시간을 밀리초 단위로 가져와 `now`라는 변수에 저장한다.
  - 환경 변수 `SK`에서 비밀 키를 가져와 `privKey`라는 변수에 저장한다. 이는 message에 서명하고 스마트 계약에 자신의 신원을 검증하는 데 사용하는 키이다.
  - `@noble/ed25519` module을 사용해 개인 키에서 공개 키를 가져오고 이를 `pubKey`라는 변수에 저장한다. 이는 스마트 계약과 사용자에게 공유하는 키이다.
  - Request body에서 response token, 비밀 키, 사용자 address를 가져와 각각 `response`, `secret`, `userAddress`라는 변수에 저장한다.
  - 디버깅을 위해 이 변수들의 값을 로그로 남긴다.
  - 사용자의 address(`0x` 접두사 제외)와 현재 시간으로 이루어진 message를 만들어 16진수 문자열로 변환하고 이를 `message`라는 변수에 저장한다.
  - `@noble/ed25519` module을 사용해 개인 키로 message에 서명하고 서명을 `signature`라는 변수에 저장한다.
  - `@noble/ed25519` module을 사용해 message와 공개 키로 서명을 검증하고 결과를 `isValid`라는 변수에 저장한다.
  - 디버깅을 위해 이 변수들의 값을 로그로 남긴다.
  - axios module을 사용해 비밀 키와 response token을 파라미터로 포함하는 POST request를 reCAPTCHA API로 보내고 response를 `axiosResponse`라는 변수에 저장한다.
  - Response 데이터에 success 필드가 있고 그 값이 true인지 확인한다. 그렇다면 다음 필드를 가진 JSON object를 반환한다:
    - `success`: true
    - `verificationInfo`: reCAPTCHA API의 response 데이터
    - `signature`: 숫자 배열로 변환한 서명
    - `pubKey`: 숫자 배열로 변환한 공개 키
    - `message`: 숫자 배열로 변환한 message
  - 그렇지 않으면 오류를 catch하고 다음 필드를 가진 JSON object를 반환한다:
    - `success`: false
- 다음으로 문자열을 입력으로 받아 16진수 문자열을 출력으로 반환하는 `stringToHex`라는 helper 함수를 정의한다. 이 함수는 message를 16진수 형식으로 변환하는 데 사용된다.
- 마지막으로 일부 custom middleware를 사용해 not found와 error case를 처리하고 앱을 기본 module로 export한다.

이것으로 끝이다. 🎉 reCAPTCHA를 위한 backend를 구현했다.
앱을 실행하려면 `node app.ts`를 사용하거나 TypeScript가 설치되어 있다면 `ts-node app.ts`를 사용할 수 있다.
앱을 변경할 때 자동으로 다시 시작하려면 `nodemon` 같은 도구를 사용할 수도 있다.
앱을 테스트하려면 Postman이나 curl 같은 도구를 사용해 앱에 request를 보내고 response를 확인할 수 있다.

## Frontend

reCAPTCHA를 위한 frontend를 구현하려면 사용자 인터페이스를 렌더링하고 backend 및 스마트 계약과 상호작용할 수 있는 react 앱을 만들어야 한다.
또한 `@mysten/dapp-kit-react`, `@mysten/sui`, `axios`, `react-google-recaptcha` 같은 의존성도 설치해야 한다.
이 package들은 지갑 통합, transaction 실행, HTTP request, reCAPTCHA 렌더링을 돕는다.

다음은 frontend를 만드는 단계이다:

1. `pnpm create vite recaptcha-app --template react-ts`로 새 project를 초기화한다.
2. `pnpm install --save @mysten/dapp-kit-react @mysten/sui @tanstack/react-query axios react-google-recaptcha`로 의존성을 설치한다.
3. `.env`라는 파일을 만들고 다음 환경 변수를 추가한다:
   - `VITE_reCAPTCHA_SITE_KEY`: 사이트를 reCAPTCHA에 등록할 때 Google에서 받는 site key
   - `VITE_reCAPTCHA_SECRET_KEY`: 사이트를 reCAPTCHA에 등록할 때 Google에서 받는 secret key
   - `VITE_PACKAGE_ID`: 상호작용하려는 스마트 계약의 package ID
   - `VITE_REGISTRY_ID`: 상호작용하려는 스마트 계약의 registry ID
4. `App.tsx`라는 파일을 만들고 제공한 코드를 붙여 넣는다.

```typescript title='App.tsx'
import './App.css';

import {
	ConnectButton,
	useCurrentAccount,
	useCurrentWallet,
	useDAppKit,
} from '@mysten/dapp-kit-react';
import { useMutation } from '@tanstack/react-query';
import { Transaction } from '@mysten/sui/transactions';
import { SUI_CLOCK_OBJECT_ID } from '@mysten/sui/utils';
import Axios from 'axios';
import { useEffect, useState } from 'react';
import ReCAPTCHA from 'react-google-recaptcha';

interface RecaptchaApiResponse {
	success: boolean;
	challenge_ts: string; // 챌린지가 로드된 timestamp(ISO 형식 yyyy-MM-dd'T'HH:mm:ssZZ)
	hostname: string; // reCAPTCHA가 해결된 사이트의 hostname
	signature?: Uint8Array;
	pubKey?: Uint8Array;
	message?: Uint8Array;
	'error-codes'?: any[]; // 선택 사항
}

function App() {
	const currentWallet = useCurrentWallet();
	const dAppKit = useDAppKit();
	const { mutateAsync: signAndExecuteTransaction } = useMutation({
		mutationFn: (tx: Transaction) => dAppKit.signAndExecuteTransaction({ transaction: tx }),
	});
	const currentAccount = useCurrentAccount();

	const SITE_KEY = import.meta.env.VITE_reCAPTCHA_SITE_KEY!;
	const SECRET_KEY = import.meta.env.VITE_reCAPTCHA_SECRET_KEY!;
	const packageId = import.meta.env.VITE_PACKAGE_ID!;
	const registryId = import.meta.env.VITE_REGISTRY_ID!;
	const moduleId: string = 'recaptcha';

	const [isRecaptchaValid, setRecaptchaValidation] = useState(false);

	const [verificationPassedOneTime, setVerificationPassedOneTime] = useState(false);

	const [message, setMessage] = useState(new Uint8Array());
	const [pubKey, setPubKey] = useState(new Uint8Array());
	const [signature, setSignature] = useState(new Uint8Array());

	const onChange = async (token: string | null) => {
		if (token === null) {
			setRecaptchaValidation(false);
		} else {
			const recaptchaApiResponse: RecaptchaApiResponse = await verifyToken(token);

			setRecaptchaValidation(true);

			if (!verificationPassedOneTime) setVerificationPassedOneTime(true);

			if (recaptchaApiResponse.message !== undefined) setMessage(recaptchaApiResponse.message);
			if (recaptchaApiResponse.pubKey !== undefined) setPubKey(recaptchaApiResponse.pubKey);
			if (recaptchaApiResponse.signature !== undefined)
				setSignature(recaptchaApiResponse.signature);
		}
	};

	async function verifyToken(token: string): Promise<RecaptchaApiResponse> {
		try {
			const response = await Axios.post(`https://bot-prevention-api.vercel.app/verify-token`, {
				response: token,
				secret: SECRET_KEY,
				userAddress: currentAccount?.address,
			});
			return response['data'];
		} catch (error) {
			console.log(error);
		}
		return {} as RecaptchaApiResponse;
	}

	useEffect(() => {
		// 여기서 `currentWallet`로 무언가를 할 수 있다.
	}, [currentWallet]);

	return (
		<div className="App">
			<ConnectButton />
			<div>
				<button
					disabled={!verificationPassedOneTime}
					onClick={async () => {
						const transaction = new Transaction();

						transaction.moveCall({
							target: `${packageId}::${moduleId}::interact`,
							arguments: [transaction.object(registryId), transaction.object(SUI_CLOCK_OBJECT_ID)],
						});

						console.log(
							await signAndExecuteTransaction({
								transaction: transaction,
							}),
						);
					}}
				>
					Interact
				</button>
			</div>

			<div>
				<button
					disabled={!isRecaptchaValid}
					onClick={async () => {
						const transaction = new Transaction();

						transaction.moveCall({
							target: `${packageId}::${moduleId}::verify`,
							arguments: [
								transaction.object(registryId),
								transaction.pure(signature),
								transaction.pure(pubKey),
								transaction.pure(message),
							],
						});

						console.log(
							await signAndExecuteTransaction({
								transaction,
							}),
						);
					}}
				>
					Verify
				</button>
			</div>

			<hr />
			<ReCAPTCHA sitekey={SITE_KEY} onChange={onChange} />
		</div>
	);
}

export default App;
```

5. 코드가 무엇을 하는지 살펴본다.

- 먼저 앱에 필요한 module을 import한다.
- 다음으로 `@mysten/dapp-kit-react`의 hook을 사용해 현재 지갑과 현재 계정에 접근하고 `useDAppKit`과 함께 TanStack Query를 사용해 `signAndExecuteTransaction` 함수를 가져온다. 이는 지갑에 연결하고 블록체인에서 transaction을 실행하는 데 도움을 준다.
- 그다음 site key, secret key, package ID, registry ID, module ID를 위한 몇 가지 상수를 정의한다. 이 값들은 reCAPTCHA API와 스마트 계약과 상호작용하는 데 사용된다.
- 그 후 reCAPTCHA 검증 상태, 검증 결과, backend에서 받는 message, 공개 키, 서명을 저장할 몇 가지 state 변수를 정의한다. React의 `useState` hook을 사용해 이 state 변수를 관리한다.
- 다음으로 token을 입력으로 받아 reCAPTCHA component의 변경을 처리하는 `onChange`라는 함수를 정의한다. 이 함수는 사용자가 reCAPTCHA 챌린지를 완료할 때 실행된다. 이 함수에서 따르는 단계는 다음과 같다:
  - Token이 null인지 확인한다. 그렇다면 reCAPTCHA 검증 state를 false로 설정한다.
  - 그렇지 않으면 token을 인자로 `verifyToken` 함수를 호출하고 결과를 `recaptchaApiResponse`라는 변수에 저장한다. 이 함수는 backend로 POST request를 보내고 스마트 계약과 상호작용하는 데 필요한 데이터와 검증 결과를 가져온다.
  - reCAPTCHA 검증 state를 true로 설정한다.
  - `verificationPassedOneTime` state가 false인지 확인한다. 그렇다면 true로 설정한다. 이 state는 사용자가 검증을 통과한 뒤 interact 버튼을 한 번만 활성화하는 데 사용된다.
  - `recaptchaApiResponse`에 message, `pubKey`, `signature` 필드가 있는지 확인한다. 그렇다면 response의 값으로 해당 state 변수를 설정한다.
- 다음으로 token을 입력으로 받아 reCAPTCHA API response의 promise를 반환하는 `verifyToken`이라는 함수를 정의한다. 이 함수는 backend와 통신하는 데 사용된다. 이 함수에서 따르는 단계는 다음과 같다:
  - Token, secret key, 사용자 address를 body parameter로 포함해 backend URL로 POST request를 보내려 시도한다. `axios` module을 사용해 request를 보내고 response를 `response`라는 변수에 저장한다.
  - Response의 data 필드를 reCAPTCHA API response로 반환한다.
  - 오류가 발생하면 이를 catch해서 console에 기록한다.
  - 기본 reCAPTCHA API response로 빈 object를 반환한다.
- 다음으로 React의 `useEffect` hook을 사용해 `currentWallet` state가 바뀔 때 코드를 실행한다. 이 경우에는 아무 작업도 하지 않지만 원한다면 여기에 로직을 추가할 수 있다.
- 마지막으로 앱을 렌더링하는 JSX element를 반환한다. 앱은 다음 컴포넌트로 구성된다:
  - 사용자가 지갑에 연결할 수 있게 하는 `@mysten/dapp-kit-react`의 `ConnectButton` 컴포넌트.
  - 사용자가 스마트 계약과 상호작용할 수 있게 하는 버튼. 이 버튼은 사용자가 최소 한 번은 검증을 통과하지 않으면 비활성화된다. 사용자가 이 버튼을 클릭하면 `@mysten/sui`의 새 `Transaction` object를 만들고 registry ID와 clock object ID를 인자로 받아 스마트 계약의 `interact` 함수를 호출하는 `moveCall` 액션을 추가한다. 그다음 `signAndExecuteTransaction` 함수를 사용해 블록체인에서 transaction에 서명하고 실행한다. 결과도 console에 기록한다.
  - 사용자가 자신의 신원을 스마트 계약에 검증할 수 있게 하는 버튼. 이 버튼은 사용자가 reCAPTCHA 챌린지를 통과하지 않으면 비활성화된다. 사용자가 이 버튼을 클릭하면 `@mysten/sui`의 새 `Transaction` object를 만들고 registry ID, 서명, 공개 키, message를 인자로 받아 스마트 계약의 verify 함수를 호출하는 `moveCall` 액션을 추가한다. 그다음 `signAndExecuteTransaction` 함수를 사용해 블록체인에서 transaction에 서명하고 실행한다. 결과도 console에 기록한다.
  - reCAPTCHA widget을 렌더링하는 `react-google-recaptcha`의 `ReCAPTCHA` 컴포넌트. 이 컴포넌트에는 site key와 `onChange` 함수를 prop으로 전달한다.

이것으로 끝이다. 🎉 reCAPTCHA를 위한 frontend를 구현했다.
앱을 실행하려면 `pnpm run dev`를 사용할 수 있다.
앱을 테스트하려면 브라우저를 열고 `localhost:5173` URL로 이동할 수 있다.
