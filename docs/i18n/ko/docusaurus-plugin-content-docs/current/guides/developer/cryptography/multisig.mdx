다음 단계는 multisig transaction을 생성한 다음 [Sui CLI](/references/cli.mdx)를 사용해 network에 제출하는 방법을 보여준다.
transaction은 object transfer, package publish 또는 upgrade, SUI payment 등이 될 수 있다.

TypeScript SDK를 사용해 multisig address를 만들고 multisig transaction을 생성하는 방법을 더 알아보려면 [SDK documentation](https://sdk.mystenlabs.com/typescript/cryptography/multisig)을 참조한다.

<ImportContent source="prerequisites.mdx" mode="snippet" />

이 문서는 Sui CLI, 특히 `sui client`와 `sui keytool` command에 어느 정도 익숙하다고 가정한다.
따라서 어떤 command는 별도의 맥락 설명 없이 소개될 수 있다.
참조된 command의 세부사항이 확실하지 않다면 Sui CLI documentation에서 자세한 정보를 확인한다.

object를 받으려면 작업 중인 network에 이미 존재하는 address가 하나 필요하다.
이 문서는 이 address가 현재 active address(`sui client active-address`)라고 가정하지만, 접근 가능한 어떤 address여도 괜찮다.

이 문서는 또한 active environment가 Testnet(`sui client active-env`)이라고 가정한다.
이 단계는 Devnet 또는 local network에서도 수행할 수 있지만, command를 그에 맞게 조정해야 한다.

## Executing multisig transactions

multisig를 보여주기 위해 이 문서는 Sui CLI를 사용해 multisig transaction을 설정하고 실행하는 과정을 안내한다.

### Create addresses with different schemes

먼저 나중에 instruction에서 수행할 transaction의 signer 역할을 할 address 세 개를 만든다.
`sui client new-address` command를 사용해 지원되는 세 가지 key scheme에 대한 Sui address와 public key를 생성한다.

```sh
$ sui client new-address ed25519
```
```sh
$ sui client new-address secp256k1
```
```sh
$ sui client new-address secp256r1
```

console은 각 성공적인 호출에 대해 다음과 비슷한 response를 표시한다:

```sh
╭──────────────────────────────────────────────────╮
│ Created new key pair and saved it to keystore.    │
├────────────────┬─────────────────────────────────┤
│ alias          │ <ALIAS>                         │
│ address        │ <SUI-ADDRESS>                   │
│ keyScheme      │ <KEY-SCHEME>                    │
│ recoveryPhrase │ <RECOVERY-PHRASE>               │
╰────────────────┴─────────────────────────────────╯
```

blockchain data를 다룰 때 address와 hashed value는 CLI 환경에서 다루기 번거로운 긴 string을 만든다.
후속 command에서 값을 더 쉽게 참조하고(그리고 copy and paste를 쉽게 하기 위해) 이 문서는 shell variable을 사용한다.
console response에서 받은 값을 사용해 이 address들에 대한 shell variable을 설정하되, `<SUI-ADDRESS-*>` 변수는 적절한 address로 바꾼다.

```sh
$ ADDRESS1=<SUI-ADDRESS-ED25519>
```
```sh
$ ADDRESS2=<SUI-ADDRESS-SECP256K1>
```
```sh
$ ADDRESS3=<SUI-ADDRESS-SECP256R1>
```

active address를 할당한 shell variable도 하나 더 만든다.

```sh
$ ACTIVE=<ACTIVE-ADDRESS>
```

:::tip

원한다면 shell variable에 address 대신 alias 값을 설정해도 된다.

:::

```sh
$ PKEY_1=<PUBLIC-KEY-ED25519>
```
```sh
$ PKEY_2=<PUBLIC-KEY-SECP256K1>
```
```sh
$ PKEY_3=<PUBLIC-KEY-SECP256R1>
```

### Create a multisig address

multisig를 사용해 transaction에 서명하려면 `sui keytool multi-sig-address`를 사용해 multisig address를 만들어야 한다.
multisig address는 각 개별 참여 address의 public key를 사용해 생성된다.
또한 각 address에는 유효한 signature를 만드는 데 얼마나 필요한지를 결정하는 `weight` 값이 할당된다.
합산했을 때 포함된 signature의 `weight`는 command에서 함께 설정하는 `threshold` 값 이상이어야 한다.
이 예시에서는 다음 command를 사용하며, 이는 첫 번째 두 address는 유효한 multisig를 만들기 위해 적어도 하나의 signature가 더 필요하다는 뜻이다.
마지막 address는 `threshold` 값과 같은 `3`의 weight를 가지므로 그 소유자만으로도 유효한 signature를 만들 수 있다.

```sh
$ sui keytool multi-sig-address --pks $PKEY_1 $PKEY_2 $PKEY_3 --weights 1 2 3 --threshold 3
```

response는 다음과 비슷하다:

```
╭─────────────────┬─────────────────────────────────────────────────────────╮
│ multisigAddress │  <MULTISIG-ADDRESS>                                     │
│ multisig        │ ╭─────────────────────────────────────────────────────╮ │
│                 │ │ ╭─────────────────┬───────────────────────────────╮ │ │
│                 │ │ │ address         │  <SUI-ADDRESS>                │ │ │
│                 │ │ │ publicBase64Key │  <PUBLIC-KEY>                 │ │ │
│                 │ │ │ weight          │  1                            │ │ │
│                 │ │ ╰─────────────────┴───────────────────────────────╯ │ │
│                 │ │ ╭─────────────────┬───────────────────────────────╮ │ │
│                 │ │ │ address         │  <SUI-ADDRESS>                │ │ │
│                 │ │ │ publicBase64Key │  <PUBLIC-KEY>                 │ │ │
│                 │ │ │ weight          │  2                            │ │ │
```

### Add SUI to the multisig address

SUI를 받기 전에 이전 response의 맨 위에 있는 multisig address를 `MULTISIG` shell variable에 설정한다(`<MULTISIG-ADDRESS>`를 실제 address로 대체한다).

```sh
$ MULTISIG=<MULTISIG-ADDRESS>
```

`sui client objects $MULTISIG` command를 사용하면 새로 만든 multisig address에 object가 없다는 것을 볼 수 있다.
이는 transaction을 수행하기 전에 SUI를 받아야 함을 뜻한다.
multisig account에 SUI를 받으려면 `sui client faucet` command를 사용하고 `--address` flag로 multisig address를 제공한다.
이 command를 두 번 실행하여 multisig address가 적어도 SUI 두 개를 소유하게 한다.
이 예시에서는 하나는 transfer에 사용하고 다른 하나는 gas를 지불하는 데 사용하기 위해 SUI 두 개를 사용한다.

```sh
$ sui client faucet --address $MULTISIG
```

`sui client gas` command를 사용해 이제 해당 address에 적어도 SUI 두 개가 있는지 확인한다.

```sh
$ sui client gas $MULTISIG
```

### Transfer SUI to your active address

이제 multisig address에서 object를 transfer할 차례이다.
단순화를 위해 이 예시는 multisig address가 소유한 coin 중 하나를 transfer object로 사용한다.
address의 coin 하나에 대한 object ID를 복사하고 이를 사용해 shell variable 값을 설정한다.

```sh
$ COIN=<COIN-OBJECT-ID>
```

`sui client transfer` command를 사용해 transfer를 설정한다.
`--serialize-unsigned-transaction` flag는 Base64로 인코딩된 transaction bytes를 출력한다.

<ImportContent source="info-gas-budget.mdx" mode="snippet" />

```sh
$ sui client transfer --to $ACTIVE --object-id $COIN --gas-budget <GAS-AMOUNT> --serialize-unsigned-transaction
```

console은 결과값(`<TX-BYTES-RESULT>`)을 표시하며, 이를 다른 shell variable에 할당할 수 있다.

```sh
$ TXBYTES=<TX-BYTES-RESULT>
```

### Sign the transaction with two public keys

`sui keytool sign` command를 사용해 앞서 만든 address 중 두 개로 transaction에 서명한다.

:::info

`flag || sig || pk`로 serialize하기만 하면 다른 도구를 사용해 signature를 생성할 수도 있다.

:::

```sh
$ sui keytool sign --address $ADDRESS1 --data $TXBYTES
```
```sh
$ sui keytool sign --address $ADDRESS2 --data $TXBYTES
```

command를 성공적으로 호출할 때마다 다음과 비슷한 response를 받는다.

```sh
╭──────────────┬─────────────────────────────╮
│ suiAddress   │ <ADDRESS>                   │
│ rawTxData    │ <TRANSACTION-HASH>          │
│ intent       │ ╭─────────┬─────╮           │
│              │ │ scope   │  0  │           |
│              │ │ version │  0  │           |
│              │ │ app_id  │  0  │           |
|              | ╰─────────┴─────╯           │
│ rawIntentMsg │ <INTENT-MSG-HASH>           │
│ digest       │ <DIGEST-HASH>               │
│ suiSignature │ <SIGNATURE-HASH>            │
╰──────────────┴─────────────────────────────╯
```

signature를 저장할 shell variable 두 개를 더 만들되, `<SIGNATURE-HASH-*>`는 이전 command response의 값으로 바꾼다.

```sh
$ SIG_1=<SIGNATURE-HASH-ED25519>
```
```sh
$ SIG_2=<SIGNATURE-HASH-SECP256K1>
```

### Combine individual signatures into a multisig

앞서 언급했듯 multisig는 참여 signer의 weight 합이 `threshold` 값 이상이 되도록 충분한 개별 signature로 구성되어야 한다.
`sui keytool multi-sig-combine-partial-sig` command를 사용해 ed25519 signature(`weight: 1`)와 secp256k1(`weight: 2`)를 결합한다.
command를 완성하려면 multisig address를 정의했던 모든 public key, 각 weight, threshold를 제공해야 한다.

```sh
$ sui keytool multi-sig-combine-partial-sig --pks $PKEY_1 $PKEY_2 $PKEY_3 --weights 1 2 3 --threshold 3 --sigs $SIG_1 $SIG_2
```

성공하면 console은 다음과 비슷한 메시지로 응답한다.

```sh
╭────────────────────┬──────────────────────────────╮
│ multisigAddress    │  <MULTISIG-ADDRESS>          │
│ multisigParsed     │  <MULTISIG-PARSED-HASH>      │
│ multisigSerialized │  <MULTISIG-SERIALIZED-HASH>  │
╰────────────────────┴──────────────────────────────╯
```

### Execute the transaction

`sui client execute-signed-tx`를 사용해 multisig transaction을 실행한다.
이전 response에서 받은 `multisigSerialized` 값을 shell variable로 설정한 다음, 이를 사용해 `execute-signed-tx` command를 구성한다.

```sh
$ MULTISIG_SERIALIZED=<MULTISIG-SERIALIZED-HASH>
```

```sh
$ sui client execute-signed-tx --tx-bytes $TXBYTES --signatures $MULTISIG_SERIALIZED
```

성공하면 console은 transaction detail을 응답한다.
