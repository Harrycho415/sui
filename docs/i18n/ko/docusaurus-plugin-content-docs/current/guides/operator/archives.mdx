Sui 아카이브는 Sui의 모든 transaction 데이터 이력이다. 경우에 따라 피어 노드가 최신 몇 개의 epoch보다 더 뒤처진 경우, 동기화를 통해 모든 transaction과 effects를 따라잡지 못할 수 있다. 이러한 경우, 피어 노드는 동기화에 의존하는 대신 아카이브에서 관련 정보를 다운로드하는 방식으로 폴백할 수 있다.

:::info 1.51 릴리스부터 mainnet의 기본 아카이브 버킷은 요청자 부담 모델로 전환된다. 구성 예시는 아래를 참고한다. :::

## Set up archival fallback {#set-up-archival-fallback}

노드가 지연될 때 아카이브로 폴백할 수 있도록 하려면, `fullnode.yaml` 파일에 다음을 추가한다:

<tabs groupid="object-store"><tabitem value="mainnet" label="Mainnet"></tabitem></tabs>

```yaml
state-archive-read-config:
  - ingestion-url: "https://s3.us-west-2.amazonaws.com/mysten-mainnet-checkpoints"
    # 따라잡을 때 미리 읽을 object 수
    concurrency: 5
    remote-store-options:
      - ["aws_access_key_id", <YOUR_AWS_ACCESS_KEY_ID|PATH_TO_AWS_ACCESS_KEY>]
      - ["aws_secret_access_key", <AWS_SECRET_ACCESS_KEY|PATH_TO_AWS_SECRET_ACCESS_KEY>]
```

<tabitem value="testnet" label="Testnet"></tabitem>

```yaml
state-archive-read-config:
  - ingestion-url: "https://checkpoints.testnet.sui.io"
    # 따라잡을 때 미리 읽을 object 수
    concurrency: 5
```
