Visual Studio Code용 [Move Trace Debugger](https://marketplace.visualstudio.com/items?itemName=mysten.move-trace-debug) 확장 프로그램은 Move 단위 테스트에 친숙한 debugging 인터페이스를 제공한다. 코드 실행을 단계별로 진행하고, 지역 변수 값을 추적하고, 라인 별 breakpoint를 설정하여 Move 코드가 어떻게 실행되는지 이해할 수 있다.

## Install

:::info Debugger를 사용하려면 [Move extension](./move.mdx)이 설치되어 있어야 한다. Move extension에는 Move Trace Debugger가 포함되어 있으므로 별도로 설치할 필요는 없다. 개별 설치가 필요한 드문 경우를 위해 설치 지침을 포함한다. :::

Move Trace Debugger 확장 프로그램은 Visual Studio Code Marketplace에서 사용할 수 있다. <strong>Extension</strong> 보기에서 <code>Move Trace Debugger</code>를 검색하거나 <kbd>Ctrl</kbd> + <kbd>P</kbd> 또는 <kbd>⌘</kbd> + <kbd>P</kbd> 를 누르고 `ext install mysten.move-trace-debug`를 입력한다.

또는 명령줄에서 확장 프로그램을 설치하려면 `code --install-extension mysten.move-trace-debug`를 실행한다.

## Usage

:::tip[Tracing]

Debugger를 사용하려면 `tracing` 기능 플래그가 활성화된 `sui` 바이너리가 설치되어 있어야 한다. 릴리스 tarballs, Homebrew 및 Chocolatey에 포함된 `sui` 바이너리에는 이 기능이 활성화되어 있다. 소스 코드에서 빌드하는 방법을 포함하여 자세한 내용은 [Install Sui](../../guides/developer/getting-started/sui-install.mdx)를 참조한다.

:::

Move 유닛 테스트 debugging은 두 단계로 진행된다.

**I. Generate execution traces**

1. 명령 팔레트를 연다(macOS에서는 <kbd>Shift</kbd> + <kbd>⌘</kbd> + <kbd>P</kbd> , Windows/Linux에서는 <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd> ).

2. `Move: Trace Move test execution` 명령을 실행한다.

    ![Move Trace generation in the command palette](https://github.com/Sui-Dev-Korea/sui/blob/main/docs/i18n/ko/docusaurus-plugin-content-docs/current/references/ide/images/trace_palette.png?raw=true)

3. 확장 프로그램은 필터 프롬프트를 표시한다. 특정 테스트를 대상으로 하려면 필터 문자열을 입력하고, 모든 테스트를 실행하려면 필드를 비워둔 채 <kbd>Enter</kbd> 키를 누른다.

    ![Move Trace generation filter string](https://github.com/Sui-Dev-Korea/sui/blob/main/docs/i18n/ko/docusaurus-plugin-content-docs/current/references/ide/images/filter_string.png?raw=true)

4. 생성된 추적 파일은 `traces` 디렉토리에서 찾을 수 있다.

**II. Start debugging**

1. 테스트가 포함된 Move 파일을 연다.

2. 메뉴에서 **Run** -&gt; **Start Debugging**을 선택한다.

    ![Start debugging](https://github.com/Sui-Dev-Korea/sui/blob/main/i18n/ko/docusaurus-plugin-content-docs/current/references/ide/images/start_debugging.png?raw=true)

3. 파일에 여러 테스트가 있는 경우 드롭다운 메뉴에서 특정 테스트를 선택한다.

    ![Test selection](https://github.com/Sui-Dev-Korea/sui/blob/main/i18n/ko/docusaurus-plugin-content-docs/current/references/ide/images/test_selection.png?raw=true)

## Features

현재 Move Trace Debugger는 테스트 실행 추적을 통한 기본적인 순방향 debugging을 지원한다.

현재 reverst debugging 및 watch expressions에 대한 지원은 제공되지 않는다.

### Stepping through code execution

Move Trace Debugger는 단계별 실행, 단계별 진입, 단계별 종료, 계속, 중지와 같은 표준 debugging 기능을 지원한다. 일반 코드와 Move 매크로를 단계별로 실행할 수 있다.

![Stepping through code execution](https://github.com/Sui-Dev-Korea/sui/blob/main/i18n/ko/docusaurus-plugin-content-docs/current/references/ide/images/debugger_running.png?raw=true)

### Tracking variable values

Move Trace Debugger는 기본 데이터 유형, Move 구조체 및 참조의 값을 표시하는 기능을 지원한다.

현재 debugger는 변수에 대한 watch point 설정을 지원하지 않는다.

![Variable values](https://github.com/Sui-Dev-Korea/sui/blob/main/i18n/ko/docusaurus-plugin-content-docs/current/references/ide/images/variables.png?raw=true)
