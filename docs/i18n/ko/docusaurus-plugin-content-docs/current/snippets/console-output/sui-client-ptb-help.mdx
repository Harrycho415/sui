
```sh
programmable transaction block을 빌드, 미리보기, 실행한다. 셸에 따라 배열이나 기타 전달 값에
따옴표를 사용해야 할 수 있다. 이 명령의 핵심 기능 사용 예시는 --help를 사용해 확인한다.

Usage: sui client ptb [OPTIONS]

Options:
      --assign <NAME> <VALUE>
          이후 PTB에서 사용할 수 있도록 값을 변수 이름에 할당한다.
           이름만 제공하면 마지막 transaction 결과가 해당 이름에 바인딩된다.
           이름과 값을 함께 제공하면 해당 이름이 그 값에 바인딩된다.
          
          예시:
           --assign MYVAR 100
           --assign X [100,5000]
           --split-coins gas [1000, 5000, 75000]
           --assign new_coins # 이전 transaction 결과를 new_coins에 바인딩한다

      --dry-run
          실행하는 대신 PTB 드라이 런을 수행한다.

      --dev-inspect
          실행하는 대신 PTB dev-inspect를 수행한다.

      --gas-coin <ID>
          사용할 가스 코인의 object ID이다. 지정하지 않으면 요청된 gas-budget 잔액 이상을 가진
          첫 번째 가스 코인을 찾아 사용하려고 시도한다.

      --gas-budget <MIST>
          이 PTB의 선택적 가스 예산이다(MIST 단위). 가스 예산이 제공되지 않으면 도구는 먼저
          가스 비용을 추정하기 위해 드라이 런을 수행한 다음 transaction을 실행한다.
          추가 드라이 런 호출로 인해 성능에 소량의 비용이 발생함에 유의해야 한다.

      --gas-price <MIST>
          이 PTB의 선택적 가스 가격이다(MIST 단위). 지정하지 않으면 참조 가스 가격을 RPC에서
          가져온다.

      --gas-sponsor <ADDRESS>
          이 PTB의 선택적 가스 스폰서이다. 지정하지 않으면 sender를 가스 스폰서로 사용한다.

      --make-move-vec <TYPE> <[VALUES]>
          동일 타입의 n개 값을 받아 벡터를 구성한다. object가 아니거나 빈 벡터인 경우에는
          타입 태그를 지정해야 한다.
          
          예시:
           --make-move-vec <u64> []
           --make-move-vec <u64> [1, 2, 3, 4]
           --make-move-vec <std::option::Option<u64>> [none,none]
           --make-move-vec <sui::coin::Coin<sui::sui::SUI>> [gas]

      --merge-coins <INTO_COIN> <[COIN OBJECTS]>
          제공된 coin으로 N개의 coin을 병합한다.
          
          예시:
           --merge-coins @coin_object_id [@coin_obj_id1, @coin_obj_id2]

      --move-call <PACKAGE::MODULE::FUNCTION> <TYPE_ARGS> <FUNCTION_ARGS>
          함수에 move call을 수행한다.
          
          예시:
           --move-call std::option::is_none <u64> none
           --assign a none
           --move-call std::option::is_none <u64> a

      --split-coins <COIN> <[AMOUNT]>
          지정된 금액 배열에 따라 coin을 N개의 coin으로 분할한다.
          
          예시:
           --split-coins gas [1000, 5000, 75000]
           --assign new_coins # split-coins 명령 결과를 변수 new_coins에 바인딩한다
           --split-coins @coin_object_id [100]

      --transfer-objects <[OBJECTS]> <TO>
          지정된 address로 object를 전송한다.
          
          예시:
           --transfer-objects [obj1, obj2, obj3] @address
                      
           --split-coins gas [1000, 5000, 75000]
           --assign new_coins # 다음에 사용하기 위해 split-coins 결과를 new_coins에 바인딩한다
           --transfer-objects [new_coins.0, new_coins.1, new_coins.2] @to_address

      --publish <MOVE_PACKAGE_PATH>
          Move 패키지를 게시한다. 패키지가 있는 폴더를 입력으로 받는다.
          
          예시:
           --move-call sui::tx_context::sender
           --assign sender
           --publish "."
           --assign upgrade_cap
           --transfer-objects sender "[upgrade_cap]"

      --upgrade <MOVE_PACKAGE_PATH>
          Move 패키지를 업그레이드한다. 패키지가 있는 폴더를 입력으로 받는다.

      --preview
          transaction을 실행하는 대신 PTB 명령을 미리본다.

      --tx-digest
          transaction을 실행하는 대신 digest를 출력한다.

      --sender <SENDER>
          활성 address 대신 이 address를 sender로 설정한다.

      --serialize-unsigned-transaction
          transaction을 실행하는 대신 서명되지 않은 transaction 데이터의 bcs 바이트를 base64
          인코딩으로 직렬화한다.

      --serialize-signed-transaction
          transaction을 실행하는 대신 서명된 transaction 데이터의 bcs 바이트를 base64
          인코딩으로 직렬화한다.

      --summary
          짧은 요약(digest, 실행 상태, 가스 비용)만 표시한다. 모든 transaction 데이터와 실행
          effects가 필요한 경우 이 플래그를 사용하지 않는다.

      --warn-shadows
          동일한 변수 이름이 여러 번 선언될 때 shadow 경고를 활성화한다. 기본값은 비활성이다.

      --json
          명령 출력값을 json 형식으로 반환한다.

  -h, --help
          도움말을 출력한다('-h'로 요약 확인)
```
