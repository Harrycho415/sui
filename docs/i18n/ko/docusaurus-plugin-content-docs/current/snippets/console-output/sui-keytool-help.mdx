
```sh
Sui keystore 도구

Usage: sui keytool [OPTIONS] <COMMAND>

Commands:
  update-alias
          기존 alias를 새 alias로 갱신한다. 새 alias가 제공되지 않으면 무작위 alias가 생성된다
  convert
          Hex 또는 Base64 private key를 새 형식(33바이트 flag || private key를 Bech32로
          인코딩한 "suiprivkey" 시작 형식)으로 변환한다. Hex private key 형식의 가져오기와
          내보내기는 Sui Wallet과 Sui CLI Keystore 모두에서 deprecated되었다. 키를 Sui
          Keystore로 가져오려면 `sui keytool import`를 사용한다
  decode-or-verify-tx
          Base64 인코딩 transaction bytes가 주어지면 구성 요소를 디코딩한다. signature가
          제공되면 transaction에 대해 signature를 검증하고 결과를 출력한다
  decode-multi-sig
          Base64 인코딩 MultiSig signature가 주어지면 구성 요소를 디코딩한다. tx_bytes가 전달되면
          multisig를 검증한다
  generate
          키 체계 플래그 {ed25519 | secp256k1 | secp256r1}로 새 키페어를 생성하며 선택적
          파생 경로를 사용한다. 기본값은 ed25519의 경우 m/44'/784'/0'/0'/0', secp256k1의 경우
          m/54'/784'/0'/0/0, secp256r1의 경우 m/74'/784'/0'/0/0이다. 단어 길이는 { word12 |
          word15 | word18 | word21 | word24}를 사용할 수 있으며 지정하지 않으면 기본값은
          word12이다
  import
          입력 니모닉 구문 또는 "suiprivkey"로 시작하는 Bech32 인코딩 33-byte `flag ||
          privkey`, 키 체계 플래그 {ed25519 | secp256k1 | secp256r1}, 선택적 파생 경로(기본값:
          ed25519는 m/44'/784'/0'/0'/0', secp256k1은 m/54'/784'/0'/0/0, secp256r1은
          m/74'/784'/0'/0/0)를 사용해 Sui CLI Keystore에 새 키를 추가한다. 단어 길이 12,
          15, 18, 21, 24의 니모닉 구문을 지원한다. --alias 플래그로 키의 alias를 설정한다.
          alias를 제공하지 않으면 도구가 자동 생성한다
  export
          Sui CLI Keystore에서 지정된 키 ID의 private key를 `suiprivkey`로 시작하는 Bech32
          인코딩 문자열로 출력한다
  list
          sui.keystore에 있는 모든 키를 Sui address, Base64 인코딩 public key, 키 체계
          이름으로 나열한다
  load-keypair
          제공된 파일 경로의 내용을 읽는다. 허용 형식은 [enum SuiKeyPair](33-byte `flag ||
          privkey`의 Base64 인코딩) 또는 `type AuthorityKeyPair`(`privkey`의 Base64 인코딩)이다.
          이 명령은 account keypair를 Base64 인코딩 `flag || privkey`로 출력하고, network
          keypair, worker keypair, protocol keypair를 Base64 인코딩 `privkey`로 출력한다
  multi-sig-address
          MultiSig Sui Address로 변환한다. Base64로 인코딩된 모든 public keys `flag || pk`
          목록을 전달한다. 예시 public keys는 `keytool list`를 참조한다
  multi-sig-combine-partial-sig
          참여 signature 목록(Base64 인코딩 `flag || sig || pk`), threshold, MultiSig
          address를 정의하는 모든 public key 목록과 각 가중치 목록을 제공한다. 유효한 MultiSig
          signature와 sender address를 반환한다. 결과는 `sui client execute-signed-tx`의
          signature 필드로 사용할 수 있다. 모든 signature 가중치의 합은 threshold 이상이어야 한다
  multi-sig-combine-partial-sig-legacy
          
  show
          제공된 파일 경로의 내용을 읽는다. 허용 형식은 [enum SuiKeyPair](33-byte `flag ||
          privkey`의 Base64 인코딩) 또는 `type AuthorityKeyPair`(`privkey`의 Base64 인코딩)이다.
          이 명령은 Base64 인코딩 public key와 키 체계 플래그를 출력한다
  sign
          sui keystore에 있는 지정 address(또는 alias)의 private key로 signature를 생성한다.
          모든 signature는 BCS 직렬화 transaction bytes 자체의 Base64 인코딩과 그 intent로
          구성된 [struct IntentMessage]에 커밋한다. intent가 없으면 기본값을 사용한다
  sign-kms
          AWS KMS를 활용해 signature를 생성한다. key-id를 전달해 Amazon KMS로 메시지에
          서명하고 base64 pubkey를 전달한다. pem에서 PubKey를 생성하려면
          MystenLabs/base64pemkey를 사용한다. 모든 signature는 BCS 직렬화 transaction bytes
          자체의 Base64 인코딩과 그 intent로 구성된 [struct IntentMessage]에 커밋한다.
          intent가 없으면 기본값을 사용한다
  unpack
          Base64 인코딩 33-byte `flag || privkey`의 [enum SuiKeyPair]를 입력으로 받는다.
          현재 디렉터리에 address를 파일명으로 하여 keypair를 파일로 출력하고, Sui address,
          Base64 인코딩 public key, 키 체계, 키 체계 플래그를 출력한다
  zk-login-sign-and-execute-tx
          max_epoch가 주어지면 OAuth url을 생성하고, 사용자에게 id_token이 포함된 redirect를
          붙여넣게 한 다음, salt server와 prover server를 호출하고, 테스트 transaction을 만들고,
          ephemeral 키로 서명한 뒤 직렬화된 zkLogin signature로 조합해 실행한다
  zk-login-enter-token
          위 명령에서 token 붙여넣기가 동작하지 않는 경우(Facebook)용 우회 명령이다.
          여기에 필요한 모든 입력은 위 명령에서 출력된다
  zk-login-sig-verify
          zkLogin signature가 주어지면 유효한 경우 파싱한다. `bytes`가 제공되면
          `intent_scope`에 따라 TransactionData 또는 PersonalMessage로 파싱한다. 최신으로
          가져온 JWK를 기반으로 zkLogin signature를 검증한다. 요청 예시: sui keytool
          zk-login-sig-verify --sig $SERIALIZED_ZKLOGIN_SIG --bytes $BYTES --intent-scope 0 --network devnet
          --curr-epoch 10
  zk-login-insecure-sign-personal-message
          TESTING ONLY: 고정 ephemeral 키와 테스트 issuer의 JWT token을 생성한다. 지정된
          데이터와 max epoch에 대한 zklogin signature를 생성한다. 예: sui keytool
          zk-login-insecure-sign-personal-message --data "hello" --max-epoch 5
  help
          이 메시지 또는 지정된 하위 명령의 도움말을 출력한다

Options:
      --keystore-path <KEYSTORE_PATH>  
      --json                           명령 출력값을 json 형식으로 반환한다
  -h, --help                           도움말을 출력한다
  -V, --version                        버전을 출력한다
```
