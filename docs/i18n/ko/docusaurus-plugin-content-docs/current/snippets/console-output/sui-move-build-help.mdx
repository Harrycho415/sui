
```sh
Usage: sui move build [OPTIONS]

Options:
  -p, --path <PACKAGE_PATH>
          명령을 실행할 기준 패키지 경로이다
      --with-unpublished-dependencies
          아직 게시되지 않은 의존성 패키지의 내용을 포함한다(base64로 bytecode를 덤프할 때만
          관련 있다)
      --dump-bytecode-as-base64
          base64로 출력할지 여부이다
      --ignore-chain
          [주로 테스트용이며 운영 환경에는 권장하지 않음] bytecode를 Base64로 덤프할 때 활성
          chain에 맞춰 패키지를 특수화하지 않는다. 이를 통해 네트워크 연결이나 활성 환경 없이도
          빌드를 진행할 수 있지만, 의존성 address를 자동으로 결정할 수는 없다
  -d, --dev
          'dev' 모드로 컴파일한다. 이 플래그가 설정되면 'dev-addresses'와
          'dev-dependencies' 필드를 사용한다. 이 플래그는 특정 값으로 설정되지 않은 named
          address를 노출하는 패키지 개발에 유용하다
      --generate-struct-layouts
          true이면 이 패키지 모듈이 선언한 `entry` 함수에 전달되는 모든 struct 타입에 대해
          struct layout schema를 생성한다. 이 layout schema는 transaction 인자와 events의
          직렬화/역직렬화를 가능하게 하기 위해 클라이언트(예: TypeScript SDK)에서 소비할 수 있다
      --test
          'test' 모드로 컴파일한다. 'dev-addresses'와 'dev-dependencies' 필드를 'tests'
          디렉터리 코드와 함께 사용한다
      --doc
          패키지 문서를 생성한다
      --disassemble
          생성된 bytecode의 디스어셈블리와 bytecode map(디스어셈블된 bytecode용 source map)을
          함께 저장한다
      --install-dir <INSTALL_DIR>
          컴파일 산출물 설치 디렉터리이다. 기본값은 현재 디렉터리이다
      --force
          모든 패키지 재컴파일을 강제한다
      --fetch-deps-only
          의존성 repo만 MOVE_HOME으로 가져온다
      --skip-fetch-latest-git-deps
          최신 git 의존성 가져오기를 건너뛴다
      --default-move-flavor <DEFAULT_FLAVOR>
          패키지 config에 지정되지 않은 경우 move 컴파일 기본 flavor이다
      --default-move-edition <DEFAULT_EDITION>
          패키지 config에 지정되지 않은 경우 move 컴파일 기본 edition이다
      --dependencies-are-root
          설정되면 의존성 패키지를 root 패키지로 처리한다. 특히 의존성 패키지의 경고 억제를
          제거한다
      --silence-warnings
          설정되면 컴파일러 경고를 무시한다
      --warnings-are-errors
          설정되면 경고를 오류로 처리한다
      --json-errors
          설정되면 오류를 JSON으로 보고한다
      --no-lint
          `true`이면 linters를 비활성화한다
      --lint
          `true`이면 추가 linters를 활성화한다
      --mode <MODE>
          임의 모드이며 컴파일 중 사용자 정의 `#[mode(<MODE>)]` annodations를 활성화하거나
          필터링하는 데 사용된다
  -h, --help
          도움말을 출력한다
  -V, --version
          버전을 출력한다
```
