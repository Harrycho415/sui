`BalanceManager` shared object는 다양한 자산에 대한 모든 잔액을 보관한다. 거래를 수행하려면 `BalanceManager`와 `TradeProof`의 조합을 [pool](./design.mdx#pool)에 전달해야 한다. `TradeProof`는 `BalanceManager` 소유자가 직접 생성하거나 어느 `TradeCap` 소유자든 생성하는 두 가지 방식 중 하나로 생성된다. 소유자는 equivocation 위험 없이 `TradeProof`를 생성할 수 있다. `TradeCap` 소유자는 owned object이기 때문에 `TradeProof`를 생성할 때 equivocation 위험을 감수한다. 일반적으로 고빈도 거래 엔진이 기본 소유자로서 거래를 수행한다.

Swap을 제외한 DeepBookV3와의 모든 상호 작용에는 입력 중 하나로 `BalanceManager`가 필요하다. 주문이 매칭되면 자금이 `BalanceManager`로 입금되거나 `BalanceManager`에서 출금된다. 모든 pool 사이에서 하나의 `BalanceManager`를 사용할 수 있다.

## API

`BalanceManager`가 노출하는 다양한 public 함수는 다음과 같다.

### Create a BalanceManager

`new()` 함수는 `BalanceManager` hot potato(ability가 없는 struct)를 생성한다. 이 함수 호출은 `share`와 결합해야 하며 그렇지 않으면 transaction이 실패한다. Transaction을 deposit 호출과 결합해 BalanceManager를 생성하고 예치한 뒤 공유하는 과정을 하나의 transaction에서 수행할 수 있다.

### Mint a TradeCap

`BalanceManager`의 소유자는 `TradeCap`을 mint해 다른 주소로 전송할 수 있다. 해당 주소가 이를 수신하면 이 `BalanceManager`로 주문을 제출할 수 있는 capability를 갖게 된다. 그러나 해당 주소 owner는 자금을 예치하거나 인출할 수는 없다. `BalanceManager`에 대해 할당할 수 있는 `TradeCap`, `WithdrawCap`, `DepositCap`의 총 최대 개수는 `1000`이다. 이 한도에 도달하면 새 cap을 mint하기 전에 기존 cap 하나 이상을 revoke해야 한다. `revoke_trade_cap`을 사용해 `DepositCap`과 `WithdrawCap`을 revoke할 수도 있다.

<ImportContent source="packages/deepbook/sources/balance_manager.move" mode="code" org="MystenLabs" repo="deepbookv3" fun="mint_trade_cap,revoke_trade_cap" />

### Mint a DepositCap or WithdrawCap

`BalanceManager`의 소유자는 `DepositCap` 또는 `WithdrawCap`을 민트하여 다른 address로 전송할 수 있다. 해당 address가 이를 수신하면 `BalanceManager`에 자금을 예치하거나 인출할 수 있는 capability를 갖게 된다. 그러나 해당 address 소유자는 거래를 실행할 수는 없다. `BalanceManager`에 대해 할당할 수 있는 `TradeCap`, `WithdrawCap`, `DepositCap`의 총 최대 개수는 `1000`이다. 이 한도에 도달하면 새로운 cap을 민트하기 전에 기존 cap 하나 이상을 revoke해야 한다.
