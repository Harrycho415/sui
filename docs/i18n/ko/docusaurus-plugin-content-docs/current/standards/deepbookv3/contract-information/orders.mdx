사용자는 지정가 주문 또는 시장가 주문을 생성하고, 주문을 수정하고, 주문을 취소할 수 있다. `BalanceManager`는 주문을 처리하기 위해 필요한 자금을 보유해야 한다. DeepBookV3는 4가지 주문 옵션과 3가지 자체 매칭 옵션을 제공한다. `pay_with_deep` 플래그를 `true`로 설정하면 거래 수수료가 DEEP 토큰으로 지불된다. `pay_with_deep` 플래그를 `false`로 설정하면 거래 수수료가 입력 토큰으로 지불된다.

사용자는 기존 주문을 수정하여 크기를 줄이거나 만료 시간을 낮추거나 둘 다 할 수 있다. 사용자는 주문 크기를 늘리거나 만료 시간을 늘리기 위해 주문을 수정할 수 없다. 그렇게 하려면 원래 주문을 취소하고 새 주문을 해야 한다.

사용자는 단일 주문을 취소하거나 모든 주문을 취소할 수 있다.

### Place limit order

지정가 주문을 한다. 수량은 베이스 자산 기준이다. 현재 버전에서는 <code>pay_with_deep</code>가 true여야 하므로 수수료가 DEEP 토큰으로 지불된다.

주문을 하기 전에 <code>BalanceManager</code> 호출로 <code>TradeProof</code>를 생성하는 것을 결합해야 한다.

<ImportContent source="packages/deepbook/sources/pool.move" mode="code" org="MystenLabs" repo="deepbookv3" fun="place_limit_order" nocomments />

### Place market order

시장가 주문을 한다. 수량은 베이스 자산 기준이다. 매수 주문의 경우 <code>MAX_PRICE</code>, 매도 주문의 경우 <code>MIN_PRICE</code>로 <code>place_limit_order</code>를 호출한다. DeepBookV3는 체결되지 않은 수량에 대해 주문을 취소한다.

<ImportContent source="packages/deepbook/sources/pool.move" mode="code" org="MystenLabs" repo="deepbookv3" fun="place_market_order" nocomments />

### Modify order

<code>order_id</code>와 <code>new_quantity</code>가 주어진 주문을 수정한다. 새 수량은 원래 수량보다 작고 체결된 수량보다 많아야 한다. 주문은 아직 만료되지 않았어야 한다.

<code>modify_order</code> 함수는 아무것도 반환하지 않는다. transaction이 성공하면 수정이 성공한 것으로 간주한다.

<ImportContent source="packages/deepbook/sources/pool.move" mode="code" org="MystenLabs" repo="deepbookv3" fun="modify_order" nocomments />

### Cancel order

주문을 취소한다. 주문은 <code>balance_manager</code>가 소유해야 한다. 주문은 주문서와 <code>balance_manager</code>의 미체결 주문에서 제거된다. <code>balance_manager</code> 잔액은 주문의 남은 수량으로 업데이트된다.

수정과 유사하게 <code>cancel_order</code>는 아무것도 반환하지 않는다. DeepBookV3는 <code>OrderCanceled</code> 이벤트를 발생시킨다.

<ImportContent source="packages/deepbook/sources/pool.move" mode="code" org="MystenLabs" repo="deepbookv3" fun="cancel_order" nocomments />
